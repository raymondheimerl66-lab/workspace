# Operating Procedures — Unternehmens-Setup

## 1. Eingang: Ray's Input
**Kanäle:**
- Telegram (Chat + Sprachmemos)
- Mission Control Dashboard (direkte Eingaben)
- GitHub Issues

**Prozess:**
1. Input empfangen
2. Transkribieren (bei Audio)
3. In Mission Control → Tasks (Inbox)
4. Kategorisieren & Priorisieren (P1/P2/P3)
5. Agent zuweisen

## 2. Verarbeitung: Team-Workflow

### Harvey (Chief of Staff)
- Entscheidet: Wer macht was
- Eskaliert: Was braucht Ray's Approval
- Berichtet: Status & Ergebnisse

### CodeBot
- Baut: Features, Fixes, Deployments
- Reviewt: Code-Qualität
- Deployed: Zu Railway

### ResearchBot
- Recherchiert: Markt, Konkurrenz, Trends
- Füllt: Radar-Sektion
- Berichtet: Weekly Market Update

### ContentBot
- Schreibt: Docs, Copy, Content
- Pflegt: Memory, Docs-Sektion
- Erstellt: SOPs, Templates

### Pulse
- Monitored: System-Health
- Alertet: Bei Problemen
- Loggt: Alle Aktivitäten

## 3. Ausgang: Delivery an Ray
- Klare Status-Updates
- Fertige URLs (GitHub, Railway)
- Zusammenfassungen (nicht Rohdaten)
- Nächste Schritte (Action Items)

## 4. Kontinuierliche Pflege
- Alle Änderungen = sofort in Mission Control
- Jeder Task = assigned + deadline
- Jede Entscheidung = dokumentiert in Council
- Jede Woche = Team-Review
